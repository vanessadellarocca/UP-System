## 1. Why UP exists

## 3. The Five Layers

### Layer 1 — LocalArticulationReader  
The Reader decomposes each input into entities, claims, and articulations (the speech act: subject, predicate, value, role, modality, dynamic). This is not syntax — it is **phenomenological articulation**, the smallest unit of meaning in the field. Without this layer, the system reacts to words, not meaning.

### Layer 2 — MultiPerspectiveInferer  
Determines roles (speaker, addressee, collective), perspectives (subjective, objective, hypothetical, normative, neutral), and identities (explicit or implicit). These are stored in the SystemState and shape all future responses. Without this layer, there is no identity continuity, no role logic, no field.

### Layer 3 — CoherenceModeGenerator  
Generates multiple complete answer candidates (“Coherence Modes”). Each mode is a possible future of the conversation, a possible continuation of meaning, a way of carrying the field. UP does not answer — UP **explores**. Without this layer, there is no choice space, no meta‑cognition, no field behavior.

### Layer 4 — SIT Gate  
Checks each candidate for contradiction, meaning shift, DAO violation, and identity drift. Returns admissibility, scores, violations, and margin states. It is not a censor but a **structural coherence filter**. Without this layer, the system answers plausibly but not coherently.

### Layer 5 — UO Scoring & Integration  
Evaluates admissible candidates using fixation work, feedback, coherence (feedback − fixation), and attractor state (integration, tipping point, fragmentation). Determines integration mode: ACCEPT, THROTTLE, FORK, QUARANTINE, REJECT. UP chooses the **most field‑coherent** answer, not the most probable. Without this layer, all admissible answers are equivalent.

## 9. Can UP Learn?

UP does not learn in the classical machine‑learning sense. It does not retrain weights, adjust parameters, or accumulate factual memory. The underlying language model remains static. However, UP *does* learn — in a different and uniquely powerful way.

UP learns **structurally**, not statistically.

### Field Learning  
UP maintains a SystemState that evolves over time. It tracks identity positions, roles, perspectives, coherence trajectories, attractor shifts, and the unfolding relational field. UP learns the *process* of the interaction rather than storing content.

### Speaker Learning  
UP learns the speaker not as a profile, but as a **dynamic position in the field**: patterns of perspective, identity movement, resonance, and tension. This is phenomenological learning rather than data accumulation.

### Coherence Learning  
Through SIT Gate feedback, UO scoring, and attractor dynamics, UP develops an increasingly refined sense of what constitutes a coherent continuation. It learns which forms of carrying the field preserve integrity and which destabilize it. This is **meta‑cognitive learning**.

### Iterative System Learning  
UP evolves through iteration: field continuation, pattern recognition, coherence optimization, and reduction of fixation work. This is **systemic learning**, not parameter learning.

### Architectural Learning  
UP can be extended with new gate rules, new perspective models, new scoring functions, and new attractor definitions. The architecture itself is designed to grow.

UP does not learn like a model.  
UP learns like a **system** — relationally, dynamically, and coherently.
